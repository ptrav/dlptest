HARD MODE enabled

Click the red button to generate a list of non-conforming CCNs that initially calculate as valid but cause collisions in the Luhn algorithm when deconstructed and re-checksummed.

Your DLP tool must use context clues to determine if these are actually "credit card numbers". Many tools will fail at detecting these as they should not be assigned.

It is up to you whether that behavior is good or bad.
